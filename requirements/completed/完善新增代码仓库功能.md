# 需求：完善新增代码仓库功能

## 背景
当前仓库管理已具备基础的列表与新增能力，但与既定规范存在差距：部分字段缺失、校验不完整、同步/删除链路不闭环、以及数据库字段长度与接口校验不一致，导致新增仓库后无法稳定支撑后续版本更新与发布流程。

## 目标
- 新增/编辑仓库时可录入并保存：仓库名称、仓库地址（cloneUrl）、初始版本号（可选）、默认分支（可选）、monoRepo 标记
- 支持分页查询与关键字筛选，筛选范围覆盖 name/cloneUrl（不引入项目维度筛选）
- 支持手动同步仓库统计信息，并在 GitLab 未配置时返回明确错误
- 前端表单校验、提示与加载/空态符合规范

## 验收标准
- [x] 创建/更新仓库时校验：name<=128、cloneUrl<=512、defaultBranch<=128（当 defaultBranch 为空时使用默认值 main）
- [x] 创建/更新仓库时支持传入 initialVersion（可选）；传入时保存为仓库初始版本号
- [x] 列表分页参数使用 1-based page/size，返回 page/size/total 元信息
- [x] keyword 能匹配 name/cloneUrl（大小写不敏感）
- [x] 删除不存在仓库返回业务错误码；删除成功返回 success=true
- [x] 手动同步在未配置 GitLab baseUrl/token 时返回 GITLAB_SETTINGS_MISSING，且不修改仓库数据
- [x] 手动同步成功后更新统计字段与 lastSyncAt，并可在页面看到最新数据

## 需求讨论要点
- 初始版本号来源：支持手工录入；也支持“同步初始版本号”从仓库文件自动提取
- 默认分支策略：默认 main；允许用户在新增/编辑时覆盖
- 同步口径：区分“同步统计（分支/MR/门禁）”与“同步初始版本号”
