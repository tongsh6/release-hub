# 需求：发布协调日历视图

> 优先级：高
> 状态：进行中（月视图已完成，周视图进行中）

## 背景

ReleaseHub 作为"发布协调平台"，核心价值之一是降低"协调成本"。当前系统缺少可视化的时间线/日历视图，无法直观看到：
- 发布窗口的时间分布
- 多个发布窗口之间的时间冲突
- 发布节奏的全局概览

## 目标

提供发布窗口的日历视图，让用户能够：
1. 按月/周查看发布窗口的时间分布
2. 直观识别时间冲突
3. 快速导航到发布窗口详情

## 验收标准

- [x] 月视图：显示当月所有发布窗口
- [ ] 周视图：显示当周发布窗口详情（含精确时间）
- [ ] 月/周视图切换按钮
- [x] 状态颜色区分：DRAFT/PUBLISHED/CLOSED 用不同颜色
- [x] 冻结标识：冻结的发布窗口有特殊标记
- [x] 点击跳转：点击日历项可跳转到详情页
- [x] 今日标记：当前日期有明显标识
- [x] 响应式：适配不同屏幕宽度

## 技术方案

### 组件库选择

使用 Element Plus 内置能力 + CSS Grid 自实现简易日历，避免引入额外依赖。

### 后端 API

复用现有 API：
- `GET /api/v1/release-windows/paged?page=1&size=100` 获取发布窗口列表
- 前端按 `plannedReleaseAt` 字段进行日历渲染

### 前端实现

1. 修改 `src/views/calendar/CalendarView.vue`，新增 `viewMode` 状态与周视图 computed
2. 添加 i18n 文案（`calendar.week`、`calendar.month`、`calendar.weekOf`）
