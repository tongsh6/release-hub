# 需求：版本更新功能增强

> 来源：`add-version-updater` 变更中延后的功能
> 优先级：低（Phase 2）

## 背景

版本更新核心功能已在 `add-version-updater` 中实现并归档。当前先从“分支推导服务”子项开始落地，目标是让版本校验接口在推导目标版本的同时，返回与 BranchRule 一致的目标分支。

## 本阶段范围（优先）

### 分支推导服务

- **描述**：根据 ReleaseWindow 与 BranchRule 自动推导目标分支（derivedBranch）
- **范围**：先落地在 `POST /api/v1/release-windows/{id}/validate` 链路
- **依赖**：BranchRule 后端实现（已具备）
- **非目标**：本阶段不改造批量版本更新执行接口入参

## 后续子项（暂不实现）

### 1. 多模块 Maven 版本一致性处理
- **描述**：处理 Maven 多模块项目的版本一致性问题
- **场景**：父 POM 版本更新后，子模块版本自动同步

### 2. 冲突检测增强
- **描述**：版本号冲突、分支冲突的自动检测
- **场景**：多仓库同时更新时的冲突预警

### 3. 前端 E2E 测试
- **描述**：版本更新功能的端到端测试

## 验收标准（本阶段）

- [x] 版本校验接口返回 `derivedBranch`
- [x] 推导分支与 BranchRule 规则一致（不一致时返回明确错误）
- [x] 现有版本推导能力（derivedVersion）保持兼容
- [x] 后端测试覆盖成功与失败分支推导场景
