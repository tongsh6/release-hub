## 完整目标蓝图

### 最终行为

用户在 BranchRule 管理页面可以：
1. 创建 TEMPLATE（模板匹配）或 REGEX（正则匹配）类型的分支规则
2. 为规则设置 scope（全局/项目/子项目级别）
3. 启用/禁用规则而不需要删除
4. 用测试分支名验证规则是否匹配
5. 查看规则的状态、scope 和生效范围

### 完整范围

- Domain：BranchRule 模型升级（新增 REGEX 类型、scope、enabled 字段、description）
- Application：BranchRuleUseCase 扩展（enable/disable/test 方法）
- Infrastructure：JPA 迁移（新增字段的 Flyway 脚本）
- Interfaces：新增 enable/disable/test API 端点
- Bootstrap：集成测试
- Frontend：激活 `modules/branchRule.ts`，升级 `BranchRuleList.vue`
- 文档：API 文档新增 BranchRule 端点

### 非目标

- 不实现分支规则的自动应用/强制执行（Phase 2）
- 不实现规则的历史变更审计（Phase 2）
- 不实现规则导入/导出

### 架构形态

- Domain：BranchRule 聚合根扩展字段（type: TEMPLATE|REGEX, scope, enabled, description）
- Application：BranchRuleUseCase 新增 enable/disable/test
- Infrastructure：JPA 字段映射 + Flyway 迁移
- Port/Adapter：不变，BranchRulePort 已有 CRUD

### 风险

- 现有 ALLOW/BLOCK 数据迁移到 TEMPLATE/REGEX：ALLOW → TEMPLATE，BLOCK → REGEX（语义上 ALLOW 表示"允许的分支模式"→ 模板匹配；BLOCK 表示"禁止的分支模式"→ 更接近正则排除）
- 前端 `modules/branchRule.ts` 与实际后端 API 可能有差异，需要对齐
- 旧前端 `branchRuleApi.ts` 需要替换为 `modules/branchRule.ts`
