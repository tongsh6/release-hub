## 关键技术决策

1. **合并 Provider 类**：Pact 的 `@Provider` 类会验证 pact 文件中全部 interaction，`@State` 方法必须在同一类中才能被找到。因此 5 个独立 PactIT 合并为单个 `AllPactIT.java`。

2. **安全开关**：Pact provider 测试发送无认证 HTTP 请求（consumer contract 不含 Authorization header），在 SecurityConfig 中增加 `security.pact-verification` 属性开关，仅在 test profile 启用。

3. **Pact 独立 Vitest 配置**：`vitest.pact.config.ts` 使用 `environment: node` + `pool: forks` + `fileParallelism: false`，避免 Pact native library 与 jsdom 的模块解析冲突。
