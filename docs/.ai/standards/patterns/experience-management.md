# 模式：三层经验管理（Experience Management）

> 从 ReleaseHub Experience Indexer + Session Summarizer 实践提炼的可复用模式。

## 问题

AI 每次从零开始，不吸取教训：
- 同样的坑反复踩，每次都要重新发现
- 历史决策没有记录，新任务无法参考
- 经验散落在聊天记录中，无法系统检索

## 方案

建立**三层经验管理体系**：索引 → 文档 → 摘要，让 AI 在执行任务前自动检索历史经验。

```
三层结构：

┌─────────────────────────────────┐
│  索引层（experience-index.md）     │  ← AI 任务前自动检索
│  关键词 → 经验条目的快速索引        │
├─────────────────────────────────┤
│  文档层（lessons/）                │  ← 匹配后加载详细内容
│  每个经验的完整记录                 │
├─────────────────────────────────┤
│  摘要层（summaries/）              │  ← 任务后自动生成
│  会话摘要 + 经验候选提取            │
└─────────────────────────────────┘
```

## 核心机制

### 1. 索引层：快速检索入口

经验索引文件维护所有经验的结构化索引，格式标准：

```markdown
### 问题类别：{category}

- **问题**：{问题描述}
- **解决方案**：{解决方案摘要，一句话}
- **相关文件**：`{经验文档路径}`
- **标签**：`tag1`, `tag2`, `tag3`
- **相关度关键词**：keyword1, keyword2, keyword3
```

**索引设计要点**：
- 关键词要全面：包含中英文、同义词、相关术语
- 解决方案要简洁：一句话概括核心思路
- 标签用于分类，关键词用于模糊匹配

### 2. 文档层：详细经验记录

每个经验对应一个独立文档，包含完整的上下文：

```markdown
# {经验标题}

## 背景
[问题发生的上下文]

## 问题
[具体问题描述]

## 解决方案
[详细解决方案]

## 关键决策
[为什么选择这个方案，考虑了哪些替代方案]

## 注意事项
[容易忽略的细节、边界情况]
```

### 3. 摘要层：自动沉淀

每次任务完成后，Session Summarizer Skill 自动：
1. 生成会话摘要（关键决策、变更点、验证方式）
2. 提取经验候选（值得沉淀的经验点）
3. 建议是否创建新的经验文档和索引条目

## 相关度计算

Experience Indexer 使用加权匹配计算相关度：

```
相关度 = (关键词匹配度 × 0.4) + (领域匹配度 × 0.3) + (任务类型匹配度 × 0.3)
```

检索策略优先级：
1. **精确匹配**：关键词完全匹配
2. **领域匹配**：优先返回同领域的经验
3. **任务类型匹配**：根据任务类型筛选（如 bugfix 优先返回错误处理经验）
4. **模糊匹配**：关键词部分匹配或同义词匹配

## 自动沉淀流程

```
任务执行完成
  ↓
Session Summarizer 生成摘要
  ├─→ 保存会话摘要到 summaries/YYYY-MM-DD-{topic}.md
  └─→ 提取经验候选
       ├─→ 有价值 → 创建 lessons/{filename}.md
       │           → 更新 experience-index.md
       └─→ 无新经验 → 仅保留会话摘要
```

## 参考实现

ReleaseHub 的经验管理实现：

| 组件 | 路径 | 职责 |
|------|------|------|
| Experience Indexer Skill | `.ai/skills/skill-experience-indexer.md` | 检索经验索引，计算相关度 |
| Session Summarizer Skill | `.ai/skills/skill-session-summarizer.md` | 生成会话摘要，提取经验候选 |
| 经验索引 | `.ai/summaries/experience-index.md` | 索引层：11 个经验类别 |
| 经验文档 | `context/experience/lessons/` | 文档层：详细经验记录 |
| 会话摘要 | `.ai/summaries/` | 摘要层：会话记录 |

## 可定制点

| 定制点 | 说明 | ReleaseHub 的选择 |
|--------|------|-------------------|
| 索引格式 | 经验条目的字段和结构 | 问题/方案/文件/标签/关键词 |
| 分类体系 | 经验的分类维度 | 技术经验 + 业务经验 |
| 相关度权重 | 匹配算法的权重分配 | 关键词 40% + 领域 30% + 类型 30% |
| 沉淀触发 | 何时自动提取经验 | 每次任务完成后 |
| 存储位置 | 经验文档的存放目录 | `context/experience/lessons/` |

## 实证数据

ReleaseHub 项目中的经验管理效果：
- 索引条目：11 个经验类别（技术 6 + 业务 5）
- 会话摘要：2+ 个归档摘要
- 覆盖领域：状态管理、版本策略、DDD 架构、数据库迁移、API 设计、前端测试、发布流程、分支管理、迭代管理

## 适用场景

此模式适用于：
- 项目生命周期较长，会反复遇到类似问题
- 团队或 AI 需要积累领域知识
- 希望边际成本递减，避免重复踩坑

不适用于：
- 一次性项目或原型验证
- 极小项目，经验管理的维护成本大于收益
