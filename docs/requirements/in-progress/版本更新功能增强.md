# 需求：版本更新功能增强

> 来源：`add-version-updater` 变更中延后的功能
> 优先级：低（Phase 2）

## 背景

版本更新核心功能已在 `add-version-updater` 中实现并归档；分支推导、多模块 Maven 子项、冲突检测增强均已完成并归档。当前阶段主体工作已收尾。

## 本阶段范围（优先）

### 冲突检测增强（已完成）

- **描述**：版本号冲突、分支冲突、合并冲突预检、跨仓库版本一致性的自动检测
- **范围**：新增 `ConflictDetectionAppService` + REST API + 前端冲突面板
- **实现**：`ConflictDetectionController` (POST/GET /api/v1/release-windows/{id}/conflicts)
- **归档**：`openspec/changes/archive/` (待建)

### 多模块 Maven 版本一致性处理（已完成）

- **描述**：父 POM 版本更新后，子模块 `parent.version` 自动同步；对于显式与父版本一致的子模块 `project.version` 同步更新
- **范围**：先落地在 `POST /api/v1/release-windows/{id}/execute/version-update` 与批量接口链路中 Maven 更新器能力
- **依赖**：`MavenVersionUpdaterAdapter` 现有单文件更新实现
- **非目标**：本阶段不引入 Maven Model API，不处理跨仓库依赖版本联动

## 后续子项（暂不实现）

### 1. 前端 E2E 测试
- **描述**：版本更新功能的端到端测试

- [x] 多模块 Maven 场景有自动化测试覆盖
