# 需求：分页标准化

## 背景
当前多个列表模块存在分页契约不一致（page 是否 0/1 based、total 字段位置不统一）、部分列表仍采用前端分页与全量拉取，导致分页结果不稳定、性能与体验不可控。

## 目标
- 统一服务端分页请求与响应契约（1-based `page`、`size`；响应包含 `page/size/total`）
- 列表默认使用服务端分页；保留非分页 list 仅用于下拉或小数据量场景
- 前后端分页行为一致，避免"看起来翻页了但数据不对"

## 验收标准
- [x] 所有主列表分页请求统一使用 1-based `page` 与 `size`
- [x] 分页响应统一返回 `page.page/page.size/page.total`
- [x] 前端列表组件分页切换与后端返回一致，不出现重复/漏数据
- [x] 相关模块的自动化测试覆盖分页与筛选关键路径
