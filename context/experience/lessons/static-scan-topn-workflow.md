# 实现后静态扫码与 TopN 修复留痕

## 背景

AI 工具完成代码实现后，如果只运行测试而不主动做静态扫描，容易遗漏安全、架构、可维护性和格式类问题；如果只口头说明“已扫描”，后续审查也无法追溯处理过程。

## 约定

统一使用：

```bash
scripts/dev/static-scan-topn.sh 10
```

生成报告：

```text
.ai/reports/static-scan/<timestamp>/summary.md
```

报告必须保留：

- 原始扫描日志
- TopN 问题清单
- 每项处理方式
- 每项处理结果
- 复扫证据
- 未解决风险

## 执行原则

1. TopN 默认取 `10`；小任务可降为 `5`，但必须写明原因。
2. 安全漏洞、架构违规和阻塞级错误不得因为 TopN 截断而忽略。
3. 跳过某项时，必须说明“不修复”的工程理由，而不是简单标记忽略。
4. 修复后必须复扫，至少覆盖被修复项所在模块或文件。
5. 最终交付必须引用报告路径和 TopN 处理结论。
